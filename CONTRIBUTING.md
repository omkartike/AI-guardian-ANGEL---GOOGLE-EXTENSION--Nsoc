## 🚀 Getting Started

1. Fork the repository
2. Clone your fork:

   ```bash
   git clone https://github.com/your-username/AI-guardian-ANGEL---GOOGLE-EXTENSION--.git
   ```
3. Navigate into the project:

   ```bash
   cd AI-guardian-ANGEL---GOOGLE-EXTENSION--
   ```

## ⚙️ Project Setup

This project is a Chrome Extension.

To run it locally:

1. Open Chrome and go to:

   ```
   chrome://extensions/
   ```
2. Enable **Developer Mode** (top right)
3. Click **Load unpacked**
4. Select the project folder

The extension will now be loaded in your browser.

## 🌿 Branching Strategy

Always create a new branch for your changes:

```bash
git checkout -b feature/your-feature-name
```

Examples:

* `feature/navbar-improvement`
* `fix/login-bug`
* `docs/contributing-guide`

---

## 🧑‍💻 Code Guidelines

* Follow the existing project structure
* Write clean, readable, and maintainable code
* Avoid unnecessary changes
* Keep functions modular and reusable
* Maintain consistency in UI design
* Comment important logic where needed

## 📝 Commit Message Convention

Use clear and meaningful commit messages:

Examples:

* `feat: add scroll-to-top button`
* `fix: resolve dropdown click issue`
* `docs: add contributing guidelines`

---

## 🔁 Pull Request Process

1. Push your branch:

   ```bash
   git push origin feature/your-feature-name
   ```
2. Open a Pull Request on GitHub
3. Add:

   * A clear and descriptive title
   * Detailed explanation of changes
   * Screenshots (if UI changes are made)
4. Link the related issue (e.g., `Fixes #issue_number`)

---

## 🐞 Reporting Issues

When reporting issues, include:

* Clear description of the problem
* Steps to reproduce
* Expected vs actual behavior
* Screenshots (if applicable)

---

## 💡 Best Practices

* Keep pull requests small and focused
* Test your changes before submitting
* Follow feedback from maintainers
* Be respectful in discussions
* Avoid duplicate issues or PRs
